1. Introduccion + Stack Tecnologico

El sistema implementa una aplicacion web para costear platos de restaurante. Permite registrar materia prima, recetas, demanda mensual, planilla, costos indirectos, activos, gastos y calcular el Estado de Resultados unitario por plato.

Stack tecnologico real

CapaTecnologiaUso
BackendFlaskRutas, formularios, calculos y renderizado Jinja2.
Base de datosSQLitePersistencia local en restaurant.db.
TemplatesJinja2Vistas HTML extendidas desde base.html.
EstilosCSS vanillaDiseño propio en static/css/style.css.
DeployVercel PythonEntrada Flask por app.py y dependencias en requirements.txt.
El repositorio conserva archivos Vite/Replit, pero la aplicacion productiva de costeo es Flask/Jinja2.

5. Modulo Ingredientes + Conversiones

Descripcion

Administra materia prima, costo de compra, unidad de compra, unidad de medida, rendimiento util y conversiones para calcular costo real por receta.

Rutas Flask

MetodoURLFuncionDescripcion
GET/ingredientesingredientesLista ingredientes y conversiones.
POST/ingredientes/nuevoingrediente_nuevoCrea ingrediente y conversion.
POST/ingredientes/editar/<iid>ingrediente_editarActualiza costo, unidad y rendimiento.
POST/ingredientes/eliminar/<iid>ingrediente_eliminarElimina ingrediente y dependencias.

Funciones de calculo

def convertir_unidad(cantidad, unidad_origen, unidad_destino, rendimiento=None)
def mp_por_plato(db, plato_id)

Formula

rendimiento_pct = rendimiento / equivalencia
costo_unitario_real = costo_compra / (equivalencia * rendimiento_pct)
costo_mp_item = costo_unitario_real * cantidad_base

Validaciones

  • Solo crea si nombre tiene valor.
  • Convierte numericos con float().
  • Si rendimiento no llega, usa equivalencia para preservar calculos.

Template asociado

ingredientes.html - formulario, tabla y acceso al kardex.

Relaciona platos con ingredientes, cantidades y unidades de uso. Es la base para calcular materia prima unitaria.

7. Modulo Proyeccion de Demanda

Descripcion

Guarda la cantidad mensual y precio referencial por plato. La cantidad mensual se usa para distribuir CIF, gastos y depreciacion.

Rutas Flask

MetodoURLFuncionDescripcion
GET/proyeccionproyeccionMuestra demanda mensual y participacion.
POST/proyeccion/guardarproyeccion_guardarActualiza o inserta proyeccion.

Funciones de calculo

def calcular_pct_participacion(db):
    pct = cantidad_plato / cantidad_total

Formula

ingreso = cantidad_mensual * precio_referencial
pct_participacion = cantidad_mensual_plato / suma(cantidad_mensual)

Validaciones

  • Convierte cantidad a entero y precio a decimal.
  • Ignora filas con valores no numericos.

Template asociado

Descripcion

Productividad define tiempo teorico por plato. Estudios de tiempo permite registrar tareas reales y elegir si el costo MOD usa minutos teoricos o suma real observada.

Rutas Flask

MetodoURLFuncionDescripcion
GET/productividadproductividadParametros MOD por plato.
POST/productividad/guardarproductividad_guardarGuarda tiempos y productividad.
GET/estudios-tiempoestudios_tiempo_indexRedirige al primer plato.
GET/estudios-tiempo/<plato_id>estudios_tiempoMuestra tareas observadas.
POST/estudios-tiempo/<plato_id>/nuevoestudio_tiempo_nuevoRegistra una tarea.

10. CIF + Depreciacion

Descripcion

El CIF unitario incluye costos indirectos manuales, depreciacion mensual de activos y planilla MOI distribuida por participacion de demanda.

Rutas Flask

MetodoURLFuncionDescripcion
GET/costos-indirectoscostos_indirectosCIF agrupado por categoria.
POST/costos-indirectos/nuevocif_nuevoAgrega CIF.
POST/costos-indirectos/editar/<cid>cif_editarActualiza CIF.
GET/activos-fijosactivos_fijosDetalle de depreciacion.

Funciones de calculo

def calcular_depreciacion_mensual(db)
def depreciacion_por_plato(db, pct, cantidad_mensual)
def cif_por_plato(db, plato_id)

Formula

depreciacion_mensual = (valor_adquisicion - valor_residual) / vida_util_anos / 12
CIF_item = (monto * pct_participacion) / cantidad_mensual
CIF = CIF_manual + depreciacion_unitaria + MOI_unitaria

11. Modulo Gastos

Descripcion

Administra gastos administrativos, ventas y financieros. GA incluye planilla ADMIN; GV incluye planilla VENTAS; GF se distribuye como gasto financiero manual.

Rutas Flask

MetodoURLFuncionDescripcion
GET/POST/gastos-admin...gastos_admin, ga_*CRUD gastos administrativos.

GET/POST/gastos-financieros...gastos_financieros, gf_*CRUD gastos financieros.

Funciones de calculo

def gastos_por_plato(db, tabla, plato_id):
    gasto_unitario = (monto * pct) / cantidad
    if tabla == "gastos_admin": agrega Planilla personal ADMIN
    if tabla == "gastos_ventas": agrega Planilla personal VENTAS

Validaciones

  • Solo inserta cuando concepto tiene valor.
  • Las lineas automaticas de planilla no son editables en gastos.

13. Modulo Kardex

Descripcion

Registra movimientos de inventario por ingrediente: entradas, salidas y ajustes. Mantiene saldo de cantidad y saldo valorizado.

Rutas Flask

MetodoURLFuncionDescripcion
GET/kardex/<ing_id>kardexLista movimientos.
POST/kardex/<ing_id>/nuevokardex_nuevoRegistra movimiento.

Formula

ENTRADA: saldo_cantidad += cantidad; saldo_valor += cantidad * costo_unitario
SALIDA: saldo_cantidad = max(0, saldo_cantidad - cantidad)
        costo_prom = saldo_valor / saldo_cantidad si saldo_cantidad > 0
AJUSTE: saldo_cantidad = cantidad; saldo_valor = cantidad * costo_unitario
La salida calcula el costo promedio despues de restar cantidad. Esto funciona para saldos positivos, pero conviene revisarlo si se requiere promedio ponderado estricto antes de salida.

Template asociado

kardex.html - movimientos y saldo final por ingrediente.

15. Validaciones y Manejo de Errores

ModuloValidacion implementadaManejo
ProyeccionCantidad y precio numericos.Filas invalidas se omiten.
IngredientesNombre obligatorio.Solo inserta si hay nombre.
RecetasPlato, ingrediente y cantidad > 0.Evita insercion incompleta.
ProductividadProductividad entre 1 y 100.Flash de error y continua con otros platos.
ResultadoPlato existente y proyeccion disponible.Redireccion a dashboard/proyeccion con mensaje.
SQLiteForeign keys activadas.PRAGMA foreign_keys = ON.
MigracionColumnas detectadas por PRAGMA.ALTER TABLE solo si falta columna.

Mensajes de usuario

El sistema usa flash() con categorias success, warning y danger, renderizadas en base.html.

No se implementa autenticacion de usuarios ni roles. El acceso es abierto dentro del despliegue.

17. Requisitos de Instalacion + Despliegue

Requisitos locales

  • Python 3.x.
  • Flask instalado desde requirements.txt.
  • SQLite incluido con Python.
cd artifacts/restaurant-app
pip install -r requirements.txt
python app.py

Despliegue en Vercel

  • Root Directory: artifacts/restaurant-app.
  • Framework: Flask u Other.
  • Build Command: vacio si Vercel respeta Flask; no usar Vite para esta app.
  • Output Directory: vacio.
  • vercel.json redirige todas las rutas a /app.py.

Limitacion importante

SQLite en Vercel sirve para demostracion. Para produccion con escritura persistente conviene migrar a una base administrada.
